    Introduction

    Welcome to the Postit API! You can use our API to access Postit API endpoints, with which you can create and access user and group data in our database.

    The API is implemented in Javascript(Nodejs). You can view code examples in the dark area to the right.

    This API documentation page was created with Slate.

    Authentication

    Some endpoints (routes) in this API are protected. They require a token which you can get when you signup as a new user or login as existing user.

    To authorize requests to protected endpoints, set the x-auth token in the request header like this::

    {
  "x-auth": "YOUR_AUTH_TOKEN"
}
    You must replace YOUR_AUTH_TOKEN with your personal API key.

    Users

    Some routes here are protected. You would need to set the x-auth token in the request header to access them

    Signup new user

    Allows new users create accounts and access protected rooutes

    Request:

    {
  "username": "daenerys"
  "email": "theunburnt@asoiaf.com"
  "password": "dracarys" 
}

    Response body:

    {
  "message": "welcome daenerys",
  "user": {
    "id": 5,
    "username": "daenerys",
    "email": "theunburnt@asoiaf.com",
    "about": "New to Postit",
    "createdAt": ,
    "udatedAt": ,
  }
}

    Request

    • Endpoint: POST: api/user/signup
    • Body (application/json)

    Response

    • Status: 201: Created
    • Body (application/json)
    • Returns x-auth token in header

    Request password reset

    Existing users can request a password reset and get a reset email

    Sample Request

    {
  "email": "no-one@example.com"
}

    Sample Response

    {
    "message": "An email with reset instructions has been sent to no-one@example.com"
}

    Request

    • Endpoint: POST: api/user/signup
    • Body (application/json)

    Response

    • Status: 200: OK
    • Body (application/json)
    • Sends recovery link to provided email if registered

    Reset Password

    Users who have already gotten a reset email, can go ahead to change their passwords, using the reset link as authentication. The reset hash expires after a few hours and can only be used once.

    Sample request: 

    {

}

    Sample response:

    {

        "id": 2,

        "about": "I am NO ONE",
        "createdAt": "2017-09-08T18:26:12.944Z",
        "updatedAt": "2017-09-09T22:52:30.814Z"
    }
}

    Request

    • Endpoint: POST: api/user/signup
    • Body (application/json)

    Response

    • Status: 201: Created
    • Body (application/json)
    • Returns x-auth token in header

    Groups

    All group routes are protected, and so require authentication. You must set the x-auth token in the header to access them

    Create new group

    A registered user can create a new group with this endpoint. The name property is required.

    Sample Request:

    {
  "name": "ASOIAF",
  "description": "For fans and characters of A Song of Ice and Fire",
  "type": "private"
}

    Sample Response

    {
  "message": "Group created",
  "group": {
    "id": 1,
    "name": "ASOIAF",
    "description": "For fans and characters of A Song of Ice and Fire",
    "createdBy": "ray",
    "type": "private",
    "updatedAt": "2017-09-17T15:07:32.369Z",
    "createdAt": "2017-09-17T15:07:32.369Z"
  }
}

    Request

    • Endpoint: POST: api/group
    • Body (application/json)
    • Requires Authentication

    Response

    • Status: 201: Created
    • Body (application/json)

    Add user to group

    A group user can add another user to a group, with the username. You must belong to the group to interact with it.

    Sample Request:

    {
  "username": "jonsnow"
}

    Sample Response:

    {
  "message": "jonsnow added to group"
}

    Request

    • Endpoint: POST: api/group/:groupid/user
    • Body (application/json)
    • Requires Authentication

    Response

    • Status: 201: Created
    • Body (application/json)

    Remove user from group

    A group creator can remove users from a group.

    Sample Request:

    {
  "username": "sansa"
}

    Sample Response:

    {
  "message": "sansa removed from group"
}

    Request

    • Endpoint: DELETE: api/group/:groupid/user
    • Body (application/json)
    • Requires Authentication
    • Only the group creator can make successfull requests to this endpoint

    Response

    • Status: 201: Created
    • Body (application/json)

    Search users that are not members of a group

    Request

    • Endpoint: GET: api/group/:groupid/notmembers?username=bran&limit=10&offset=0
    • Requires: Authentication

    URL parameters

    Parameter Description
    groupid ID of the group

    Query Parameters

    Parameter Default Description
    username "" Search query.
    limit Sets the number of results to return. If empty, returns the whole result.
    offset 0 Sets the number of results to skip.

    Response

    • Status: 200: OK
    • Body (application/json)

    Change group info

    Edit the info for an existing group. Group name is required. If description is absent, it is set to null. If no type is specified, it is left unchanged.

    Sample JSON request: 

    {
  "name": "Beautiful intellect",
  "description": "Love your mind"
}

    Sample JSON response: 

    {
    "message": "Group info successfully updated",
    "group": {
        "id": 2,
        "name": "Beautiful intellect",
        "createdBy": "ray",
        "description": "Love your mind",
        "type": "public",
        "createdAt": "2017-09-14T07:41:01.684Z",
        "updatedAt": "2017-09-24T15:30:18.084Z"
    }
}

    Request

    • Endpoint: PATCH: api/group/:groupid/info
    • Body (application/json)
    • Requires Authentication

    Response

    • Status: 201: OK
    • Body (application/json)

    Leave group

    This endpoint allows a registered user who has been added to a group to leave it.

    Sample response: 

    {
  "message": "robb has left the group"
}

    Request

    • Endpoint: POST: api/group/:groupid/leave
    • Body (application/json)
    • Requires Authentication

    Response

    • Status: 201: OK
    • Body (application/json)
    • Deletes the group when everyone has left

    Delete group

    This endpoint allows a group creator delete the group

    Sample response: 

    {
  "message": "Group successfully deleted"
}

### Request
- Endpoint: POST: `api/group/:groupid/remove`
- Body `(application/json)`
- Requires Authentication

### Response
- Status: `201: OK`
- Body `(application/json)`

    Errors

    This is a list of the status codes you might see when you encounter an error

    The Postit API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- You did not quite structure your request as you should have.
    401 Unauthorized -- Your x-auth access token is invalid.
    403 Forbidden -- You cannot access this routes, because you have insufficient permissions although authenticated.
    404 Not Found -- The specified group or user, or even route, could not be found.
    500 Internal Server Error -- We had a problem with our server. Try again later.
